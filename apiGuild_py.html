API путеводитель

Прикладной Програмный Интерфейс Антпула (API), дает возможность доступа пользователям управлять своим аккаунтом используя данную программу.

Ограничения по запросам

Нельзя делать более 600 запросов в течении 10 минут, в противном случае ваш IP адрес будет заблокирован.

API идентификация

Все личные API звонки требуют идентификацию. Вам нужно предоставить 3 ключа для идентификации:

  • API ключ
  • Номер
  • Подпись

API ключ

Для получения API ключа, ройдите в "API". Установите в настройках и нажмите "Генерировать ключ".

Номер

Номер это регулярное целое число. Оно должно увеличиваться с каждым вашим запросом. Более в деталях. Пример: если вы установите номер 1 в вашем 1 запросе, вы должны установить 2 в вашем втором запросе. Обычная практика использовать цифру сравнимую с вашим запросом.

Подпись

Подпись это HMAC-SHA256 закодированное сообщение включающее в себя: имя клиента и ключ API. HMAC-SHA256 код долже быть сгененрирован используя секретный ключ, который был сгенерирован вместе с API ключем. Код должен быть сконвертирован в свой шестнадцатеричное представление (64 симывола).

Пример (Python):

message = userid + api_key + nonce
signature = hmac.new(API_SECRET, msg=message, digestmod=hashlib.sha256).hexdigest().upper()

Пример (Java):

public static String apiSignature(String userId, String key, String nonce, String secret) throws Exception {
    String data = userId+key+nonce;
    Mac hmacSha256 = Mac.getInstance("HmacSHA256");
    SecretKeySpec secretKey = new SecretKeySpec(secret.getBytes(), "HmacSHA256");
    hmacSha256.init(secretKey);
    return Hex.encodeHexString(hmacSha256.doFinal(data.getBytes())).toUpperCase();
}

public static boolean checkApiSignature(String userId, String key, String nonce, String secret, String signature) throws Exception {
    String data = userId+key+nonce;
    Mac hmacSha256 = Mac.getInstance("HmacSHA256");
    SecretKeySpec secretKey = new SecretKeySpec(secret.getBytes(), "HmacSHA256");
    hmacSha256.init(secretKey);
    String signatrueTmp = Hex.encodeHexString(hmacSha256.doFinal(data.getBytes())).toUpperCase();
    if(signature.equals(signatrueTmp)) {
        return true;
    }
    return false;
}

Личные настройки

Статистика пула

POST https://antpool.com/api/poolStats.htm

Парамеры:

  • ключ - API ключ
  • номер - номер
  • подпись - подпись

Вернуть статистику пула как указано ниже:

{
"version":"1.0",
"code":0,
"message":"ok",
"data":
  {
  "poolHashrate":"0", //Pool hash rate in MH/s
  "activeWorkerNumber":"0", //Active worker number
  "poolStatus":"Alive", //Pool run status
  "networkDiff":"0", //Current network difficulty
  "estimateTime":"0", //Block estimate time in second
  "currentRound":"0", //Current run time in second
  "totalShareNumber":"0", //Total share number
  "totalBlockNumber":"0" //Total block number
  }
}

Баланс аккаунта

POST https://antpool.com/api/account.htm

Параметры:

  • ключ - API ключ
  • номер - номер
  • подпись - подпись

Возвращает балан пользователя в Биткойны. См образец ниже:

{
"version":"1.0",
"code":0,
"message":"ok",
"data":
  {
  "earn24Hours":"0.00000000",
  "earnTotal":"0.00000000",
  "paidOut":"0.00000000",
  "balance":"0.00000000"
  }
}

Хашрэйт

POST https://antpool.com/api/hashrate.htm

Params:

  • ключ - API ключ
  • номер - номер
  • подпись - подпись

Возвращает все мощи в MH/s, общее принятое и отказанное дифф одной доли. См образец ниже:

{
"version":"1.0",
"code":0,
"message":"ok",
"data":
  {
  "last10m":"0",
  "last30m":"0",
  "last1h":"0",
  "last1d":"0",
  "prev10m":"0",
  "prev30m":"0",
  "prev1h":"0",
  "prev1d":"0",
  "accepted":"0",
  "stale":"0",
  "dupelicate":"0",
  "other":"0"
  }
}

Хашрэйт рабочих

POST https://antpool.com/api/workers.htm

Параметры:

  • ключ - API ключ
  • номер - номер
  • подпись - подпись
  • включить страницу - включение странице по умолчанию, 0:отключение; 1:включение
  • страница - перейдите на страницу номер и по умолчанию это 1
  • размер страницы - количество записей на странице по умолчанию 10

Возварщает хэшрэйт рабочих в MH/s, общее принятое и отказанное дифф одной доли.

    "last1d":"0",
    "prev10m":"0",
    "prev30m":"0",
    "prev1h":"0",
    "prev1d":"0",
    "accepted":"0",

История платежей

POST https://antpool.com/api/paymentHistory.htm

Параметры:

  • ключ - API ключ
  • номер - номер
  • подпись - подпись
  • тип - тип истории платежей и выплаты по умолчанию, выплаты: история выплат; pps: PPS история; pplns: PPLNS история; p2p: P2P история
  • включить страницу - включение странице по умолчанию, 0:отключение; 1:включение
  • страница - перейдите на страницу номер и по умолчанию это 1
